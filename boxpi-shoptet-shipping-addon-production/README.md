# Boxpi Shoptet Shipping Addon — production-oriented build

Production-oriented Shoptet shipping addon for Boxpi API.

This build includes:
- Shoptet addon install callback
- OAuth access token storage per shop
- short-lived Shoptet API token exchange
- signed webhooks
- shipping request quote flow
- pickup point widget for checkout
- Postgres persistence via Prisma
- Redis + BullMQ background jobs
- Boxpi package validation and creation
- tracking sync back to the Shoptet order remarks
- Docker deployment

## Important

This repository is meant to be deployable, but you still need:
- your own Shoptet addon registration and permissions
- your own `shippingMethodCode`
- valid Boxpi credentials
- end-to-end testing on a real Shoptet shop before going live

## Architecture

1. Shoptet calls `/api/install/shoptet` with `code`
2. addon exchanges the code for an OAuth access token
3. addon registers webhooks and renews a webhook signature key
4. checkout widget loads pickup points from `/api/shipping/pickup-points`
5. widget submits a quote to `/api/shipping/quote`
6. addon updates the Shoptet shipping request
7. Shoptet sends `shippingRequest:confirmed`
8. worker creates the Boxpi shipment in the background
9. tracking is synced back to the Shoptet order remarks

## Widget setup in Shoptet

Serve this URL publicly:

```text
https://your-addon.example.com/public/boxpi-shipping.js
```

The checkout page needs to provide these values, for example through data attributes:

```html
<div data-boxpi-eshop-id="123456"></div>
<div data-boxpi-shipping-request-code="REQ123"></div>
<div data-boxpi-shipping-guid="2f3d1f1a-1234-5678-9012-abcdefabcdef"></div>
<script src="https://your-addon.example.com/public/boxpi-shipping.js"></script>
```

## Core API endpoints

- `GET /health`
- `GET /api/install/shoptet`
- `GET /api/shipping/pickup-points?eshopId=123&carrier=PACKETA-POINT-SK`
- `POST /api/shipping/quote`
- `GET /api/shipping/status/:shippingRequestCode/:shippingGuid?eshopId=123`
- `POST /api/webhooks/shoptet`
- `GET /api/shops/:eshopId`

## Background job

The worker processes confirmed shipping requests and creates Boxpi packages asynchronously.

## Security

- webhook verification uses `Shoptet-Webhook-Signature`
- raw request body is hashed with HMAC-SHA1 using the signature key generated by Shoptet
- secrets live only in `.env`
- no client-side Boxpi or Shoptet secrets are exposed

## Production checklist

- configure HTTPS
- whitelist Shoptet webhook source IP range on your reverse proxy if desired
- monitor `/health`
- set up log shipping
- back up Postgres
- run end-to-end tests on a real addon installation
- confirm the exact request payload your shipping addon needs in your Shoptet partner setup
